# 🔬 Validate Models Guide

## Що це?

Скрипт для перевірки точності збережених моделей з `all_models.json`.  
Відновлює **повну модель** (архітектуру + натреновані ваги) з `.h5` файлів та перевіряє її на валідаційних даних.

---

## 🚀 Швидкий старт

### Валідувати ВСІ моделі:
```bash
python validate_models.py
```

### Валідувати перші 10 моделей:
```bash
python validate_models.py --max-models 10
```

### Валідувати конкретне покоління:
```bash
python validate_models.py --generation 5
```

### Вказати інший файл історії:
```bash
python validate_models.py --history output/model_history/all_models.json
```

---

## 📊 Що робить скрипт?

1. **Завантажує** `all_models.json`
2. **Відновлює** повну модель (архітектуру + натреновані ваги) з `.h5` файлів
3. **Перевіряє** точність на валідаційних даних Military Vehicles
4. **Порівнює** з оригінальною збереженою точністю
5. **Зберігає** результати:
   - `output/validation_results_*.json` - детальні результати
   - `output/validation_*.log` - повний лог виконання

```json
      "difference": 0.0007,
      "status": "✅ ІДЕНТИЧНО"

    ...
  ]

```

### Файл `output/validation_*.log`:

Повний лог виконання скрипта (дублює консольний вивід):
- Завантаження даних
- Процес валідації кожної моделі
- Підсумкова статистика
- Повідомлення про помилки (якщо є)

**Корисно для:**
- Аналізу проблем при валідації
- Збереження історії перевірок
- Документування результатів для статті

## 🎯 Навіщо це потрібно?

### 1. Перевірка коректності збереження
- Підтвердити що `all_models.json` зберігає правильні дані
- Виявити помилки в логіці збереження моделей

### 2. Відтворюваність результатів
- Перевірити що модель можна відновити з JSON
- Підтвердити що результати стабільні

### 3. Діагностика проблем
- Знайти моделі з великими відмінностями
- Виявити проблеми з певними архітектурами

### 4. Науковий підхід
- Доказ воспроізводимості для статті
- Валідація експериментальних результатів

## 🔍 Інтерпретація результатів

### ✅ ІДЕНТИЧНО (`|diff| < 0.001`)
- Модель точно відтворена
- Результати надійні

### ⚠️ БЛИЗЬКО (`|diff| < 0.01`)
- Невелика різниця (нормально для нейромереж)
- Може бути через рандомізацію в evaluation

### ❌ ВІДРІЗНЯЄТЬСЯ (`|diff| >= 0.01`)
- Значна різниця - потрібна перевірка
- Можлива помилка в збереженні/відновленні

---

## 🐛 Типові помилки

### "Файл не знайдено"
```bash
# Перевір шлях до all_models.json
ls output/model_history/all_models.json

# Або вкажи правильний шлях
python validate_models.py --history path/to/all_models.json
```

### "BUILD ERROR"
- Модель не може бути відновлена з JSON
- Можлива несумісність версій або корупція даних

### Великі відмінності
- Перевір що використовуються ті ж дані (Military Vehicles)
- Можливо модель не збереглась правильно
